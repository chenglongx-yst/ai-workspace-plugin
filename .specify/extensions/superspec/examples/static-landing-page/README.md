# Example: Static Landing Page (real e2e snapshot)

This directory is a **verbatim snapshot** of an end-to-end run produced by
[`scripts/e2e-agent-claude.sh`](../../scripts/e2e-agent-claude.sh).

A real Claude Code session was driven through the full SuperSpec workflow
(7 stages, all spec-kit + superspec slash commands), and every artifact
written to disk by the agent was copied here unchanged. Nothing in this
folder is hand-edited — it shows you exactly what spec-kit + superspec
look like after a full run.

The end product (`web/index.html`) is a standalone, dependency-free
landing page for the SuperSpec project itself — open it in any browser:

```bash
open examples/static-landing-page/web/index.html
```

## Directory layout — and why it matters

```text
examples/static-landing-page/
├── CLAUDE.md                            ← agent context file (spec-kit init)
├── .specify/
│   └── memory/
│       └── constitution.md              ← project constitution (Stage 1)
├── specs/
│   └── 001-static-landing-page/         ← THIS feature's working folder
│       ├── spec.md                      ← what & why (Stage 2 + 3)
│       ├── plan.md                      ← how (Stage 4)
│       ├── data-model.md                ← Stage 4 sidecar
│       ├── research.md                  ← Stage 4 sidecar
│       ├── quickstart.md                ← Stage 4 sidecar
│       ├── tasks.md                     ← T001..T029 (Stage 5)
│       ├── progress.yml                 ← live execution state (Stage 6)
│       └── checklists/
│           ├── requirements.md          ← Stage 2
│           └── review.md                ← Stage 7
└── web/
    └── index.html                       ← the actual deliverable (Stage 6)
```

Two paths people get wrong — both are illustrated above:

- The **constitution** lives at `.specify/memory/constitution.md`, **not** at
  `memory/constitution.md` and **not** at the repo root.
- Feature folders live at `specs/NNN-<slug>/`, **not** at `.specify/specs/...`.
  Spec-kit's `create-new-feature.sh` hard-codes `SPECS_DIR="$REPO_ROOT/specs"`;
  there is no other option (unless you explicitly set `SPECIFY_FEATURE_DIRECTORY`).

## Reproducing this snapshot

```bash
# from repo root
export ANTHROPIC_API_KEY=...     # or `claude login` first
scripts/e2e-agent-claude.sh
```

The script will spin up a fresh temp workdir, run the same 7-stage chain,
and assert 23 invariants on the resulting tree. To resume from a specific
stage (useful if Anthropic returns a transient socket error):

```bash
E2E_RESUME_WORKDIR=/path/to/prior/workdir \
E2E_RESUME_FROM=5 \
  scripts/e2e-agent-claude.sh
```

## What this is **not**

- Not a runnable example project — `web/index.html` is the only thing meant
  to be opened. The other files are documentation of the *process* that
  produced it.
- Not a template — copy `templates/` from the repo root for that.
- Not regenerated by CI. It is a hand-curated snapshot; refresh it
  manually whenever the workflow changes substantially.
